## 2.2. Опис інформаційного забезпечення

Підрозділ описує інформаційне забезпечення від концептуального рівня (ER) через логічний (реляційна схема) до зовнішніх потоків даних. СУБД описано узагальнено як «реляційна СУБД з просторовими типами OGC Simple Features» – вибір конкретного засобу – у 3.1.4.

### 2.2.1. Концептуальна модель даних

Виокремлено вісім сутностей: **Profile** (код `municipal`/`investor`); **Criterion** (структура за Табл. 1.10); **PairwiseMatrixCell** – асоціативна сутність «профіль–критерій», зберігає $\tilde{a}_{ij} = (l, m, u)$ (підрозділ 1.2.4); **Location** (Табл. 1.9, геометрія WGS-84); **ExistingStation** (довідник для критеріїв насиченості); **EvaluationRun** (профіль, час, статус, вектор ваг, тривалість); **RankingItem** (пара `(EvaluationRun, Location)` з $C_i^*$, $S_i^+$, $S_i^-$); **SensitivityRecord** ($p_i(k)$, довірчі інтервали, $N$, $\delta$). Концептуальну модель наведено на рис. 2.6.

![Концептуальна (ER) модель даних у нотації Чена з елементами crow's foot. Вісім сутностей (Profile, Criterion, PairwiseMatrixCell, Location, ExistingStation, EvaluationRun, RankingItem, SensitivityRecord) з атрибутами і ключами. Відношення: Profile ↔ Criterion – M:N через PairwiseMatrixCell; Profile → EvaluationRun – 1:N; EvaluationRun → RankingItem – 1:N; EvaluationRun ◇–– SensitivityRecord – 1:1 композиція; Location → RankingItem – 1:N; ExistingStation – самостійний довідник](images/fig_2_6_er_diagram.png)

Рис. 2.6. Концептуальна (ER) модель даних системи

```plantuml
@startuml
skinparam shadowing false
skinparam defaultFontName "Times New Roman"
skinparam defaultFontSize 11
hide circle
skinparam linetype ortho
skinparam entity {
    BackgroundColor #f5f5f5
    BorderColor #333333
}

entity "Profile" as Profile {
    * id : Integer <<PK>>
    --
    code : String
    name : String
    description : Text
}

entity "Criterion" as Criterion {
    * id : Integer <<PK>>
    --
    code : String
    name : String
    unit : String
    optimizationType : Enum
    scale : Enum
}

entity "PairwiseMatrixCell" as PWM {
    * profile_id : Integer <<FK>>
    * criterion_i : Integer <<FK>>
    * criterion_j : Integer <<FK>>
    --
    l : Decimal
    m : Decimal
    u : Decimal
}

entity "Location" as Location {
    * id : Integer <<PK>>
    --
    name : String
    address : String
    district : String
    geometry : Point(WGS84)
}

entity "ExistingStation" as Station {
    * id : Integer <<PK>>
    --
    name : String
    powerKW : Decimal
    connectorType : String
    geometry : Point(WGS84)
}

entity "EvaluationRun" as Run {
    * id : Integer <<PK>>
    --
    profile_id : Integer <<FK>>
    createdAt : Timestamp
    status : Enum
    weightsVector : JSON
    executionTimeMs : Integer
}

entity "RankingItem" as Rank {
    * evaluation_id : Integer <<FK>>
    * location_id : Integer <<FK>>
    --
    rank : Integer
    closenessCoefficient : Decimal
    distanceToPositive : Decimal
    distanceToNegative : Decimal
}

entity "SensitivityRecord" as Sens {
    * evaluation_id : Integer <<PK,FK>>
    --
    iterations : Integer
    perturbation : Decimal
    stabilityMatrix : JSON
    confidenceIntervals : JSON
}

Profile ||--o{ PWM
Criterion ||--o{ PWM
Profile ||--o{ Run
Run ||--o{ Rank
Run ||--|| Sens
Location ||--o{ Rank

@enduml
```

Ключові відношення: `Profile` ↔ `Criterion` – M:N через `PairwiseMatrixCell`; `Profile` → `EvaluationRun` – 1:N; `EvaluationRun` ◇–– `SensitivityRecord` – 1:1 композиція; `Location` → `RankingItem` – 1:N.
