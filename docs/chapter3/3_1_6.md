### 3.1.6. Розгортання прототипу на хмарних платформах

Прототип системи розгорнуто у трирівневій конфігурації, що відповідає Deployment Diagram підрозділу 2.3.5: клієнтська підсистема – на платформі Vercel, серверна – на Railway, база даних – на хмарному інстансі Supabase (PostgreSQL 16 + PostGIS 3.4).

Клієнтська підсистема (React / TypeScript) публікується на Vercel через GitHub-інтеграцію: кожен коміт у гілку `main` автоматично запускає збірку через Vite і деплой на глобальну CDN-мережу Vercel. Змінна середовища `VITE_API_BASE_URL` задає URL серверного API і встановлюється окремо для середовищ `preview` і `production`. Час збірки фронтенду – близько 90 с.

Серверна підсистема (FastAPI / Python) розгортається на Railway з автоматичним виявленням Python-додатка через `Procfile` (`web: uvicorn app.main:app --host 0.0.0.0 --port $PORT`). Змінні середовища `DATABASE_URL`, `SECRET_KEY` і `ALLOWED_ORIGINS` задаються в панелі Railway. Безкоштовний план (0.5 vCPU / 512 МБ RAM) відповідає мінімальним апаратним вимогам підрозділу 1.3 для базового датасету $|A|=12$, $|C|=10$.

База даних ініціалізується через послідовне виконання Alembic-міграцій (`alembic upgrade head`) і seed-скрипту `init_data.sql` у Supabase SQL Editor. Підключення сервера до Supabase виконується за connection string з Supabase Dashboard (розділ «Project Settings → Database»), переданим у змінну `DATABASE_URL`. PostGIS активується однократно командою `CREATE EXTENSION IF NOT EXISTS postgis;`.
