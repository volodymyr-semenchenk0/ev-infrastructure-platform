### 3.1.2. Засоби та програмні технології реалізації серверної частини

Серверна підсистема реалізована на мові Python 3.12 з використанням фреймворку FastAPI 0.111, вибраного через нативну підтримку асинхронного виконання (asyncio/ASGI), автоматичну генерацію OpenAPI-специфікації та декларативну валідацію через типи Python. Структура серверного проєкту організована за принципом розподілу відповідальностей: `app/api/` – REST-ендпоінти (маршрутизація та валідація), `app/services/` – обчислювальне ядро FAHP/TOPSIS/MC, `app/models/` – ORM-моделі, `app/schemas/` – Pydantic-схеми, `app/db/` – конфігурація і сесія СУБД.

Валідація вхідних даних виконується бібліотекою Pydantic v2, яка забезпечує перевірку типів, дефолтних значень і кастомних обмежень (зокрема, значення TFN $(l, m, u)$ з умовою $0 < l \leq m \leq u$). Порушення формату матриці попарних порівнянь повертає відповідь у категорії «помилка валідації» (підрозділ 2.1.6) ще до передачі запиту сервісному шару.

Доступ до бази даних організований через SQLAlchemy 2.0 з асинхронним рушієм `asyncpg`. Для роботи з просторовими типами PostGIS використовується GeoAlchemy2, яка надає об'єктно-орієнтований інтерфейс до типів `geometry(Point, 4326)` та просторових функцій `ST_DWithin`, `ST_Within`. Міграції схеми управляються через Alembic.

Обчислювальне ядро реалізовано у трьох модулях: `services/fahp.py`, `services/topsis.py`, `services/montecarlo.py`. Бібліотека `pymcdm` використовується для кросперевірки результатів TOPSIS на тестових наборах; основна реалізація є власною – векторизованою на NumPy – для забезпечення максимальної продуктивності в циклі MC ($N = 10\,000$ ітерацій з Алгоритмом 2.2 на кожній). Ключовий фрагмент реалізації методу TOPSIS:

```python
def compute_ranking(X: np.ndarray, W: np.ndarray, types: list[str]) -> RankingResult:
    norms = np.sqrt((X ** 2).sum(axis=0))
    R = X / norms
    V = R * W
    A_pos = np.where(np.array(types) == "b", V.max(axis=0), V.min(axis=0))
    A_neg = np.where(np.array(types) == "b", V.min(axis=0), V.max(axis=0))
    S_pos = np.sqrt(((V - A_pos) ** 2).sum(axis=1))
    S_neg = np.sqrt(((V - A_neg) ** 2).sum(axis=1))
    C = S_neg / (S_pos + S_neg)
    return RankingResult(closeness=C, s_pos=S_pos, s_neg=S_neg)
```

Реалізація відтворює формули (1.10)–(1.14) підрозділу 1.2.5. Векторизація на NumPy забезпечує виконання одного виклику `compute_ranking` за менш ніж 1 мкс, що задовольняє вимогу 5 с на повний цикл MC при $N = 10\,000$ (Табл. 2.5). Обробка помилок реалізована через кастомні виключення з поверненням семантично значущих категорій відповіді: порушення $CR > 0{,}10$ у модулі FAHP ініціює виключення `InconsistentMatrixError`, яке перехоплюється на рівні REST-контролера і повертається клієнту у категорії «семантична помилка».
