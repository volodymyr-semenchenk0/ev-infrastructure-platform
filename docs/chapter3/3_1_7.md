### 3.1.7. Перелік звітних форм системи

Система формує чотири типи звітних форм, доступних для завантаження з інтерфейсу (підрозділ 3.1.5, рис. 3.7). Детальна специфікація структур і наповнення кожної форми наведена у Додатку Д.

**Звіт про ваги критеріїв** (формат JSON/CSV) — вихід FAHP-модуля: ідентифікатор профілю ОПР, вектор нормалізованих ваг $W = (w_1, \ldots, w_{10})$ з відповідністю назв критеріїв, значення $CR$ матриці суджень і статус узгодженості ($CR \leq 0{,}10$ або відмова). Використовується для аудиту та порівняння вагових схем двох профілів.

**Звіт про ранжування локацій** (JSON/CSV) — вихід TOPSIS-модуля: для кожної з $n = 12$ локацій — ранг, назва, геокоординати, $C_i^*$, $S_i^+$, $S_i^-$; плюс ідентифікатор сеансу `evaluation_run_id` і часова мітка. Сортується за спаданням $C_i^*$.

**Звіт про аналіз чутливості** (JSON/CSV) — вихід MC-модуля: матриця $P = [p_i(k)]_{12 \times 3}$ (для $k \in \{1, 3, 5\}$), 95\%-ві довірчі інтервали $[Q_{0.025}, Q_{0.975}]$ для значень $C_i^*$ трьох найвищих локацій, параметри запуску ($N$, $\delta$). Посилається на звіт про ранжування через `evaluation_run_id`.

**Порівняльний звіт двох профілів** (JSON/CSV) — вихід `RankingComparator`: зведена таблиця з рангами обох профілів для кожної локації, дельта рангів $\Delta r_i = r_i^{\text{city}} - r_i^{\text{inv}}$, значення рангової кореляції Спірмена $\rho$ та її p-значення. Ідентифікує компромісні локації (з найменшим $|\Delta r_i|$) та полярно оцінені ($|\Delta r_i| \geq 5$).
