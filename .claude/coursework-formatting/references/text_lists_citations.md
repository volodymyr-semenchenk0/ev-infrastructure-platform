# Reference: Lists, Abbreviations, and Citations

Read this reference when the user works on inline lists, footnotes, abbreviations, or any kind of citation in the body text.

## Lists

### When to use which list level

The methodology distinguishes three list types:

#### Type 1: short items "in line" with text

When list items are short (one or two words) and have no internal punctuation, write them in-line with the paragraph, separated by commas:

```
Microsoft Office включає такі компоненти: Word, Excel, Access, PowerPoint.
```

This is the default for short enumerations. **Do not use bullet points** for two-or-three-word items.

#### Type 2: enumerated items with internal punctuation

When items have internal commas or are full clauses, use one of:

- a lowercase letter with closing parenthesis: `а)`, `б)`, `в)`;
- or a dash: `–`;
- or a digit with closing parenthesis: `1)`, `2)`, `3)`.

Each item starts with a paragraph indent, in lowercase, separated by semicolons; the last item ends with a period:

```
Критерії оцінювання включають:
а) щільність населення району;
б) середній дохід у районі;
в) близькість до існуючих ЗС.
```

#### Type 3: complete sentences as list items

If each item is a full sentence (with subject and predicate), each item starts with a number, capital letter, and ends with a period:

```
Система працює в двох режимах.
1. У режимі муніципалітету ваги відображають соціальні пріоритети.
2. У режимі інвестора ваги відображають економічну окупність.
```

### Nested levels

For two-level lists:

```
а) форма і розмір клітин;
б) живий склад клітин:
   1) частини клітин;
   2) неживі включення протопластів;
в) утворення тканини.
```

The nested level is offset under the first letter of the parent line.

### Forbidden bullet styles

The methodology allows only: `–`, `а)`, `1.`, `1)`. **Do not use** `•`, `*`, `→`, `■`, or any other bullet character.

## Abbreviations and acronyms

### General rules

- Specific terminology that recurs frequently can be abbreviated. The full form must appear at first mention with the abbreviation in parentheses:

  ```
  Система підтримки прийняття рішень (СППР) призначена для...
  Метод аналізу ієрархій (Analytic Hierarchy Process, AHP) дозволяє...
  ```

- After first mention, use only the abbreviation throughout the text.

- Words that have full Ukrainian equivalents should not be abbreviated mid-sentence.

### Allowed standard abbreviations

The methodology allows these without first-mention expansion:

- `тощо` (instead of «і т. д.», «і т. ін.», «і т. п.»);
- `див.`, `рис.`, `табл.` — when followed by a number;
- `ст.`, `р.`, `тис.`, `млн`, `млрд` — with numbers (`5 тис. грн`, `2025 р.`).

### Forbidden abbreviations in body text

Never abbreviate: «так званий» (т. з.), «наприклад» (напр.), «формула» (ф-ла), «рівняння» (р-ня), «діаметр» (діам.), «сторінка» (стор. — only acceptable in citations).

«Та інші» and «і таке інше» inside a sentence are written in full.

### Initials

Between initials and surname use **non-breaking spaces**: `Т. Г. Шевченко` (not `Т.Г.Шевченко` and not `Т. Г.Шевченко`).

In MS Word: Ctrl+Shift+Space to insert a non-breaking space.

### Перелік умовних скорочень

Note: the coursework methodology does **not require** a separate "Перелік умовних скорочень" section. This is an МДР element. Do not add it to the coursework. Define abbreviations inline at first mention only.

## Citations and in-text references

### Format

Citations appear in the body text in **square brackets** containing the source number from the bibliography list:

```
[3]
[8, c. 24]                ← single source with page number
[12 – 15; 18]             ← range of sources combined
[12 – 15; 18; 21, c. 42]  ← mixed
[15, 553]                 ← number with page (alternative format used in the methodology example)
```

The methodology uses both `c.` and the bare number for pages. The form `[15, 553]` (just a comma and the page number) is shown as an example in the methodology — both styles are acceptable, but pick one and apply consistently throughout.

### Inline-reference style

When the source matters and you want to mention the author in prose:

```
Як показано у роботі [12, c. 45], метод TOPSIS забезпечує...
```

### Numbering of the bibliography

Sources are numbered sequentially starting from 1. The order is **either** alphabetical **or** by order of first appearance in the text. Pick one and stick to it.

### Minimum count

The methodology requires **at least 10 sources**. For Master's-level work targeting a quality coursework, aim for **25–30 sources**, with about 70% English-language Q1/Q2 journal articles from 2020–2026.

### Citing within figures and tables

If a figure or table is taken from another source, the citation goes:

- For a figure: in the caption, after the title — `Рис. 2.3. Архітектура системи [12, c. 45]`.
- For a table: in the heading, after the title — `Таблиця 1.1. – Класифікація методів MCDM [8, c. 24]`.

## Footnotes (Підрядкові посилання)

The methodology allows footnotes for incidental references. Format:

- Marker: superscript Arabic numeral after the relevant fragment.
- Footnote text: at the bottom of the page, separated from the body by a horizontal rule.
- Numbering: per page (not continuous through the document).
- Use sparingly. Prefer in-text citations in square brackets.

## Notes (Примітки)

The methodology distinguishes notes from footnotes. Notes are placed immediately after the text/figure/table they refer to.

- Single note: not numbered. Format: `Примітка. Ціни наведені згідно з прейскурантом постачальника.`
- Multiple notes: numbered. Format: `Примітка 1.`, `Примітка 2.`, etc.

Notes attached to a table are placed under the table's bottom border line.

Font: 12 pt, single line spacing.

## Punctuation rules quick-reference

- One space after punctuation, no space before.
- Dash (тире): use «–» (U+2013) with surrounding spaces, as in `текст – текст`. The em-dash «—» (U+2014) is not used in this coursework.
- Hyphen without spaces in compound words: `науково-технічний`.
- Кутові lapky: «text», not "text" or 'text' or „text".
- Ellipsis: single character `…`, not three periods.

## Numbers and units

- Decimal separator: comma (`3,14`), not period.
- Group thousands by spaces: `1 000 000`.
- Always non-breaking space between number and unit: `25 кВт`, `10 критеріїв`, `2025 р.`, `с. 45`.
- All metric quantities only in SI units (per ГОСТ 19.781-90).

## Common errors to fix on sight

| ❌ | ✅ |
|---|---|
| `•` or `*` as bullet | `–` or `а)` or `1)` |
| `Т.Г.Шевченко` | `Т. Г. Шевченко` (non-breaking spaces) |
| `(СППР)` not introduced before use | `Система підтримки прийняття рішень (СППР)` at first mention |
| `[3, стор. 24]` | `[3, c. 24]` (Ukrainian short form) |
| `"text"` or `'text'` | `«text»` |
| `25кВт` | `25 кВт` (non-breaking space) |
| `2025р.` | `2025 р.` |
| `5.000` (period as thousands separator) | `5 000` (space) |
| Decimal point: `3.14` | Decimal comma: `3,14` |
| Three periods `...` | Single ellipsis `…` |
| `--` or `—` (U+2014) for dash | `–` (U+2013) with spaces |
